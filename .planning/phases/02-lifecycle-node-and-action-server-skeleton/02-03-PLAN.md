  **Header:** BSD-3-Clause header + module docstring: `"""TrajectoryGrouper — groups trajectory paths into blended execution groups."""`

  **Imports:** `from movement_controller.models.trajectory_path_dto import TrajectoryPathDTO`

  **Class:** `class TrajectoryGrouper:` with no `__init__` (stateless utility).

  **`group` method:** `@staticmethod def group(paths: list[TrajectoryPathDTO]) -> list[list[TrajectoryPathDTO]]:`

  Implementation:
  1. If `paths` is empty, raise `ValueError('paths list must not be empty')`
  2. Pre-validation loop — check for duplicate `path_id` values using a `set`:
     - If `path.path_id` already in the seen set, raise `ValueError(f'Duplicate path_id: {path.path_id!r}')`
     - Add each `path_id` to the set
     - Note: `path.path_id` emptiness and `motion_type` validity are already enforced by `TrajectoryPathDTO` validators; no need to re-check them here
  3. Grouping loop — build `groups: list[list[TrajectoryPathDTO]] = []`:
     - For index `i`, path `path` in `enumerate(paths)`:
       - If `i == 0` OR `path.blend_radius <= 0`: start a new group — append `[path]` to `groups`
       - Otherwise (i > 0 AND `path.blend_radius > 0`): merge — append `path` to `groups[-1]`
  4. Return `groups`

  **Algorithm correctness (D-07 example):** Input `[t0(br=0.5), t1(br=0), t2(br=0), t3(br=0.3), t4(br=0.3), t5(br=0.3), t6(br=0)]` must produce `[[t0], [t1], [t2, t3, t4, t5], [t6]]` (4 groups, 8 feedback messages). Walk through manually:
  - i=0: t0 (br=0.5) → i==0 → new group: [[t0]]
  - i=1: t1 (br=0.0) → br<=0 → new group: [[t0], [t1]]
  - i=2: t2 (br=0.0) → br<=0 → new group: [[t0], [t1], [t2]]
  - i=3: t3 (br=0.3) → i>0 and br>0 → merge: [[t0], [t1], [t2, t3]]
  - i=4: t4 (br=0.3) → merge: [[t0], [t1], [t2, t3, t4]]
  - i=5: t5 (br=0.3) → merge: [[t0], [t1], [t2, t3, t4, t5]]
  - i=6: t6 (br=0.0) → br<=0 → new group: [[t0], [t1], [t2, t3, t4, t5], [t6]]
  This matches the expected output.

  Note: `blend_radius` on the DTO is already normalised to `>= 0.0` by the `TrajectoryPathDTO.normalise_blend_radius` validator. Negative values will appear as `0.0` by the time they reach the grouper.
