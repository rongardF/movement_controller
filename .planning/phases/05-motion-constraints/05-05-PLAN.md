## Objective

Add an integration test that verifies end-to-end propagation of planning failure when a workspace
bounding-box constraint is active and PILZ rejects the plan. The test injects a
`ConstraintConfigDTO(z_max=0.5)` directly into the existing `controller` fixture's planner service
(avoiding a second controller and action server name conflict), sets
`mock_move_group.planning_success = False` to simulate `ValidateSolution` rejecting the out-of-bounds
path, sends a goal, and asserts `result.success == False`.

Purpose: CON-01 integration verification — workspace constraint active → planning failure → error
result propagated correctly to the action client.
