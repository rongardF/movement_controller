<objective>
Lay the data-model and constructor foundations that Plans 04-02 and 04-03 build on.
PlanResultDTO gains three fields required by D-06 (trajectories, path_ids, blended).
PilzPlannerService gains a required node argument and creates the /plan_sequence_path service
client at construction time. URMovementController.on_configure wires both together and
guards against the capability not being loaded in move_group.

Purpose: Unblock parallel and sequential feature work in Wave 2 and 3 without altering any
existing runtime behaviour — the plan() method is untouched, so Phase 3 behaviour is preserved.
Output: Extended model, updated constructor, updated on_configure, updated test fixture.
</objective>
