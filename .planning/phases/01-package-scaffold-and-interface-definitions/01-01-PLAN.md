# Plan 1: Hybrid Package Build System

## Goal

Create the four package root files (`CMakeLists.txt`, `package.xml`, `setup.py`, `setup.cfg`) that establish the `ament_cmake` + `ament_cmake_python` hybrid build system with correct `rosidl` interface generation configuration and the critical `SKIP_INSTALL` workaround.

## Tasks

### Task 1-1-1: Create CMakeLists.txt with SKIP_INSTALL workaround

<wave>1</wave>
<type>execute</type>
<autonomous>true</autonomous>

<read_first>
- `.planning/phases/01-package-scaffold-and-interface-definitions/01-RESEARCH.md` — Contains the exact verified CMakeLists.txt structure including SKIP_INSTALL, manual install(DIRECTORY), ament_add_pytest_test registration, and the full rationale for the workaround
- `.planning/phases/01-package-scaffold-and-interface-definitions/01-CONTEXT.md` — D-06 confirms the 5 srv files and action/msg files that must be listed in rosidl_generate_interfaces
</read_first>

<action>
Create `CMakeLists.txt` at the package root. The file must follow this exact structure and ordering:

1. Header: BSD-3-Clause license comment block (year 2026, "Movement Controller Contributors"), then blank line, then `cmake_minimum_required(VERSION 3.8)`, then `project(movement_controller)`.

2. Compiler warnings block: `if(CMAKE_COMPILER_IS_GNUCXX OR CMAKE_CXX_COMPILER_ID MATCHES "Clang")` with `add_compile_options(-Wall -Wextra -Wpedantic)`.

3. `find_package` calls (all REQUIRED): `ament_cmake`, `ament_cmake_python`, `rosidl_default_generators`, `Python3` with `COMPONENTS Interpreter Development`, `geometry_msgs`, `shape_msgs`.

4. `rosidl_generate_interfaces(${PROJECT_NAME} ...)` listing all 7 interface files:
   - `"action/ExecuteTrajectory.action"`
   - `"msg/TrajectoryPath.msg"`
   - `"srv/AddObject.srv"` `"srv/AttachObject.srv"` `"srv/DetachObject.srv"` `"srv/RemoveObject.srv"` `"srv/ModifyAcm.srv"`
   DEPENDENCIES `geometry_msgs shape_msgs`
   **SKIP_INSTALL** — this flag MUST be present to prevent the duplicate target fatal error described in RESEARCH.md.

5. `ament_python_install_package(${PROJECT_NAME})` — installs source Python files.

6. `install(DIRECTORY "${CMAKE_CURRENT_BINARY_DIR}/rosidl_generator_py/${PROJECT_NAME}/" DESTINATION "lib/python${Python3_VERSION_MAJOR}.${Python3_VERSION_MINOR}/site-packages/${PROJECT_NAME}")` — manually installs the generated Python interface files that SKIP_INSTALL suppressed.

7. `if(BUILD_TESTING)` block with `find_package(ament_cmake_pytest REQUIRED)` and `ament_add_pytest_test(test_imports "tests/unit/test_imports.py" APPEND_ENV PYTHONPATH=${CMAKE_CURRENT_BINARY_DIR}/rosidl_generator_py TIMEOUT 60 WORKING_DIRECTORY "${CMAKE_CURRENT_SOURCE_DIR}")`.

8. `ament_export_dependencies(rosidl_default_runtime)` then `ament_package()`.

Add a comment above the `rosidl_generate_interfaces` block referencing the issue URL: `# See: https://github.com/ros2/rosidl_python/issues/141`.
</action>

<acceptance_criteria>

- `grep -c "rosidl_generator_py" CMakeLists.txt` outputs `1` (the manual install DIRECTORY line)

cmake -P CMakeLists.txt 2>&1 | grep -i "error" || echo "No cmake errors found"

# Confirm SKIP_INSTALL is present:
grep "SKIP_INSTALL" CMakeLists.txt

# Confirm all 7 interface files are registered in rosidl_generate_interfaces:

## Threat Model

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Dependency version mismatch (Jazzy-incompatible package version pulled at build time) | LOW | All deps use ROS2 Jazzy apt-versioned packages; no version pinning in package.xml (standard practice for ROS2 workspaces) |
| SKIP_INSTALL removed by future ament_cmake update | LOW | Documented in CMakeLists.txt comment with issue URL; Jazzy is an LTS release with stable ABI |
| Stale build artifacts causing false-positive build success | LOW | `colcon build` with `--symlink-install` in devcontainer always builds from source; CI should `rm -rf build/ install/ log/` before build |
| No threats with HIGH or MEDIUM severity | N/A | This plan creates only build configuration files with no network access, no user input, and no executable code |
</threat_model>
