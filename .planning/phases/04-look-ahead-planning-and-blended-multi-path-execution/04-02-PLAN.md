<objective>
Implement the look-ahead planning core inside PilzPlannerService. After this plan,
the service owns: a background thread that plans all groups sequentially via the
/plan_sequence_path ROS2 service; state propagation from group N's end-state to
group N+1's start-state; thread-safe cancellation; and a generator that blocks until
each PlanResultDTO is ready.

Purpose: MOT-03 (look-ahead: plan while execute) and MOT-04 (queue: next group ready
immediately). MOT-02 (blended trajectories) is fulfilled by sending MotionSequenceRequest
to the PILZ sequence planner for every group.
Output: Expanded PilzPlannerService with plan_all, _planning_loop, _plan_group_sequence,
_build_pose_goal_constraints, _extract_end_state, iterate_planned_trajectories, cancel.
</objective>

| PilzPlannerService ↔ /plan_sequence_path | External ROS2 service call; response must be validated before use |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | _plan_group_sequence: response.planned_trajectories could be empty | mitigate | Check `response.response.error_code.val != MoveItErrorCodes.SUCCESS` before accessing planned_trajectories; _extract_end_state accesses [-1] index — only called when result is not None (success=True path) |
| T-04-02-02 | Denial of Service | Background thread holds _plan_queue mutex during cancel | accept | Mutex hold is O(1) (clear internal deque); no blocking ops inside mutex section |
| T-04-02-03 | Denial of Service | Stale cancel_event from previous goal (Pitfall 5) | mitigate | plan_all() creates fresh threading.Event() per call (D-04); event cannot carry over |
| T-04-02-04 | Information Disclosure | path_ids in error messages | accept | Internal system; path IDs are UUID4 with no PII |
| T-04-02-SC | Tampering | No new packages installed | accept | All threading/queue/time are Python stdlib; moveit_msgs is ROS2 Jazzy built-in |
