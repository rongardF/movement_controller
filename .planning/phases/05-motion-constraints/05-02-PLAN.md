## Objective

Implement all constraint-building logic inside `PilzPlannerService`:
`_build_path_constraints()` builds the `Constraints` object for a path,
`_merge_circ_and_path_constraints()` safely merges arc definition with path constraints,
and `_generate_motion_sequence_request()` is updated to inject these per item and to honour
per-path speed/acceleration scaling. The velocity warning for unenforceable joint velocities
is also added to `set_constraints()` here to keep all PilzPlannerService changes in one plan.

Purpose: Ensures every planning request carries the active workspace, joint, and orientation
constraints without overwriting CIRC arc definitions (CON-01, CON-02, CON-03, CON-04).
Speed scaling from TrajectoryPathDTO is wired to PILZ's native scaling fields (D-06, CON-05).

Output: Updated `pilz_planner_service.py` with three new private methods, updated
`set_constraints()`, and updated `_generate_motion_sequence_request()`.

    Logic:
    1. `merged = Constraints()`
    2. `merged.name = circ.name`  — preserve 'interim' or 'center' (PILZ reads this field)
    3. `merged.position_constraints = [circ.position_constraints[0]] + path.position_constraints`

    4. `merged.joint_constraints = path.joint_constraints`
    5. `merged.orientation_constraints = path.orientation_constraints`
    6. Return `merged`.

    Update `_generate_motion_sequence_request()`:

    Replace the two hardcoded lines:
      `item.req.max_velocity_scaling_factor = 1.0`
      `item.req.max_acceleration_scaling_factor = 1.0`
    With dynamic scaling (D-06, RESEARCH.md):
      `item.req.max_velocity_scaling_factor = max(0.01, min(1.0, path_dto.cartesian_speed)) if path_dto.cartesian_speed > 0.0 else 1.0`
      `item.req.max_acceleration_scaling_factor = max(0.01, min(1.0, path_dto.acceleration)) if path_dto.acceleration > 0.0 else 1.0`

    Replace the existing CIRC path_constraints block:
      `if path_dto.motion_type == MotionTypeEnum.CIRC:`
      `    item.req.path_constraints = self._build_circ_constraints(path_dto)`
    With the merged constraint injection block:
      ```
      path_constraints = self._build_path_constraints(path_dto.tool_frame or 'tool0')
      if path_dto.motion_type == MotionTypeEnum.CIRC:
          circ_constraints = self._build_circ_constraints(path_dto)
          item.req.path_constraints = self._merge_circ_and_path_constraints(
              circ_constraints, path_constraints
          )
      else:
          item.req.path_constraints = path_constraints
      ```
    This replaces the single-line CIRC branch and adds a new else branch for LIN/PTP.
