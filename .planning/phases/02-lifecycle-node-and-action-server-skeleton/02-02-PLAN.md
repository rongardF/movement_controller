  **Step 2 — Extend `on_configure`** by adding the following lines AFTER the two `declare_parameter` calls and BEFORE the `return TransitionCallbackReturn.SUCCESS`:
  - Read the parameter: `action_server_name = self.get_parameter('action_server_name').value`
  - Create the ActionServer: `self._action_server = ActionServer(self, ExecuteTrajectory, action_server_name, execute_callback=self._execute_callback, goal_callback=self._goal_callback, callback_group=ReentrantCallbackGroup())`
  - Log at INFO: `self.get_logger().info(f'Action server created at: {action_server_name}')`

  **Step 3 — Add `_goal_callback` method** to the class:
  Signature: `def _goal_callback(self, goal: ExecuteTrajectory.Goal) -> GoalResponse:`

  Body — check in this exact order:
  1. Lifecycle state check: `if self._state_machine.current_state[0] != State.PRIMARY_STATE_ACTIVE:` — log ERROR with both the reason and the current state label `self._state_machine.current_state[1]`, return `GoalResponse.REJECT`
  2. Executing flag check (under lock): `with self._executing_lock: if self._is_executing:` — log ERROR `'Goal rejected: another goal is already executing'`, return `GoalResponse.REJECT`
  3. Structural validation: if `not goal.paths` — log ERROR `'Goal rejected: paths list is empty'`, return `GoalResponse.REJECT`
  4. Per-path validation loop — for each path in `goal.paths`:
     - if `not path.path_id` — log ERROR `f'Goal rejected: path_id is empty for a path'`, return `GoalResponse.REJECT`
     - if `path.motion_type not in ('LIN', 'PTP', 'CIRC')` — log ERROR `f'Goal rejected: invalid motion_type {path.motion_type!r}'`, return `GoalResponse.REJECT`
  5. Return `GoalResponse.ACCEPT`

  **Step 4 — Add `_execute_callback` method** to the class:
  Signature: `async def _execute_callback(self, goal_handle: ServerGoalHandle) -> ExecuteTrajectory.Result:`

  Body:
  1. Set executing flag: `with self._executing_lock: self._is_executing = True`
  2. Wrap all execution in `try: ... except Exception as e: ... finally:`
  3. In `try`:
     - Convert paths: `paths = [TrajectoryPathDTO.from_ros_msg(p) for p in goal_handle.request.paths]`
     - Group: `groups = TrajectoryGrouper.group(paths)`
     - For each group in groups:
       - `path_ids = [p.path_id for p in group]`
       - Create feedback executing: `fb = ExecuteTrajectory.Feedback()`, set `fb.status = FeedbackStatusEnum.EXECUTING.value`, set `fb.trajectory_path_ids = path_ids`, call `goal_handle.publish_feedback(fb)`
       - Create feedback completed: `fb2 = ExecuteTrajectory.Feedback()`, set `fb2.status = FeedbackStatusEnum.COMPLETED.value`, set `fb2.trajectory_path_ids = path_ids`, call `goal_handle.publish_feedback(fb2)`
     - Build result: `result = ExecuteTrajectory.Result()`, set `result.success = True`, set `result.error_message = ''`, set `result.trajectory_paths_completed = [p.path_id for p in paths]`
     - Call `goal_handle.succeed()`
     - Return `result`
  4. In `except Exception as e`:
     - Log ERROR: `self.get_logger().error(f'Execution failed: {e}')`
     - Build error result: `result = ExecuteTrajectory.Result()`, set `result.success = False`, set `result.error_message = str(e)`
     - Call `goal_handle.abort()`
     - Return `result`
  5. In `finally`:
     - Clear flag: `with self._executing_lock: self._is_executing = False`

  CRITICAL API REMINDERS:
  - Use `self._state_machine.current_state[0]` — NEVER `self.get_current_state()` (does not exist in rclpy Jazzy)
  - `goal_handle.succeed()` and `goal_handle.abort()` take no required arguments
  - The `goal_callback` parameter of `ActionServer` receives the `Goal` message directly, not a `ServerGoalHandle`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ROS2 action client → _goal_callback | Untrusted Goal message (paths, path_id strings, motion_type strings) enters here |
| _goal_callback → _execute_callback | Internal boundary; only goals that passed ACCEPT cross here |
| _execute_callback → TrajectoryGrouper | Internal; grouper receives already-validated and DTO-converted paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | DoS | _goal_callback — concurrent goals | mitigate | `_is_executing` flag under `threading.Lock` rejects concurrent goals with GoalResponse.REJECT |
| T-02-10 | DoS | _goal_callback — inactive-state flooding | mitigate | Lifecycle state check is the first check; rejects all goals when node is UNCONFIGURED or INACTIVE |
| T-02-11 | Injection | _goal_callback — motion_type string | mitigate | Whitelist check `path.motion_type not in ('LIN', 'PTP', 'CIRC')` rejects any value outside the known set; no eval or exec on the string |
| T-02-12 | Injection | _goal_callback — path_id string | mitigate | Non-empty check enforced in goal_callback; path_id is only stored and echoed back, never executed or used as file path |
| T-02-13 | Tampering | _is_executing race condition | mitigate | Both read (in goal_callback) and write (in execute_callback) are wrapped in `with self._executing_lock:` context manager |
| T-02-14 | Elevation of Privilege | self.get_current_state() AttributeError | mitigate | Forbidden — using `self._state_machine.current_state[0]` (verified API); AttributeError in goal_callback would crash instead of protecting; correct API used |
| T-02-15 | Information Disclosure | error_message in Result | accept | `str(e)` echoed in `result.error_message` for debugging; acceptable for internal robot controller network; no secrets in exception messages from pure Python grouper logic |
| T-02-SC | Tampering | pip/apt installs | accept | No new packages installed; all deps already declared in package.xml |
</threat_model>
